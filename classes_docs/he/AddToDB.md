# AddToDB - תיאור

## 1. מידע כללי
*   **שם המחלקה:** `AddToDB`
*   **סוג:** Activity (מסך)
*   **ייעוד:** מסך זה מאפשר למשתמש לצלם תמונה באמצעות מצלמת המכשיר, להוסיף הערת טקסט ולשמור את שניהם בבסיס נתונים מקומי.
*   **אינטראקציה:**
    *   מבקשת הרשאות ממערכת אנדרואיד.
    *   מפעילה את אפליקציית המצלמה של המערכת.
    *   שומרת נתונים לתוך `HelperDB` (SQLite).

## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `ivImage` | `ImageView` | הצגת התמונה שצולמה במצלמה. | `onCreate()`, `onActivityResult()` |
| `imageFilename` | `String` | שם זמני לקובץ התמונה. | לחיצה על `ivImage`, `createImageFile()` |
| `imageUri` | `Uri` | ה"כתובת" של קובץ התמונה במערכת. | `callCameraApp()`, `onActivityResult()` |
| `helperDB` | `HelperDB` | חיבור לבסיס הנתונים SQLite. | לחיצה על `bSave` |
| `db` | `SQLiteDatabase` | אובייקט לביצוע פעולות SQL (הכנסה). | לחיצה על `bSave` |
| `etComment` | `EditText` | שדה טקסט להזנת תיאור על ידי המשתמש. | `onCreate()`, לחיצה על `bSave` |
| `bSave` | `Button` | כפתור לשמירת הנתונים בבסיס הנתונים. | `onCreate()` |

## 3. מתודות המחלקה
### שם המתודה: `callCameraApp`
*   **סוג:** `private`
*   **ערך חזרה:** `void`
*   **לוגיקה:**
    1.  יצירת Intent עבור `ACTION_IMAGE_CAPTURE` (פתיחת המצלמה).
    2.  יצירת קובץ זמני עבור התמונה באמצעות `createImageFile()`.
    3.  שימוש ב-`FileProvider` כדי לקבל URI מאובטח (כתובת) לקובץ.
    4.  העברת ה-URI לאפליקציית המצלמה כדי שתדע איפה לשמור את התמונה.
    5.  הפעלת המצלמה באמצעות `startActivityForResult`.
*   **מתי נקראת:** כאשר המשתמש לוחץ על ה-`ImageView`.

### שם המתודה: `onActivityResult`
*   **סוג:** `protected`
*   **ערך חזרה:** `void`
*   **פרמטרים:**
    | שם | סוג | תיאור |
    | :--- | :--- | :--- |
    | `requestCode` | `int` | מזהה הבקשה (3 עבור מצלמה). |
    | `resultCode` | `int` | האם המשתמש צילם או ביטל. |
    | `data` | `Intent` | נתונים נוספים (לא בשימוש כאן כי אנו משתמשים ב-`imageUri`). |
*   **לוגיקה:** אם המשתמש צילם בהצלחה, המתודה מגדירה את `ivImage` להציג את התמונה באמצעות ה-`imageUri`.

### שם המתודה: `bSave.onClick` (בתוך `onCreate`)
*   **לוגיקה:**
    1.  קבלת הטקסט מה-`etComment`.
    2.  קבלת הביטמפ (התמונה) מה-`ivImage`.
    3.  המרה של התמונה למערך בתים (`byte[]`) לצורך שמירה ב-DB.
    4.  פתיחת בסיס הנתונים למצב כתיבה.
    5.  הכנסת ההערה ובתי התמונה לאובייקט `ContentValues`.
    6.  הכנסת השורה לטבלת בסיס הנתונים.
    7.  סגירת בסיס הנתונים וסיום ה-Activity.

## 4. מחזור חיים (Activity)
*   **`onCreate()`:** מבקשת הרשאות (מצלמה, אחסון) ומגדירה מאזיני ממשק.
*   **`onActivityResult()`:** מטפלת בחזרה מאפליקציית המצלמה כדי להציג את התמונה.

## 5. אינטראקציה עם הממשק (UI)
*   **`ImageView`:** משמש ככפתור לפתיחת המצלמה וכחלון תצוגה מקדימה לתמונה.
*   **`EditText`:** קולט טקסט מהמשתמש.
*   **`Button`:** מפעיל את תהליך השמירה.

## 6. אינטראקציה עם רכיבים אחרים
*   **אפליקציית המצלמה:** אפליקציית מערכת חיצונית.
*   **FileProvider:** משמש לשיתוף קבצים מאובטח בין אפליקציות.
*   **HelperDB:** מחלקה מותאמת אישית לניהול בסיס הנתונים.

## 7. לוגיקה כללית של המחלקה
1.  המשתמש מאשר הרשאות.
2.  המשתמש לוחץ על האייקון של התמונה -> המצלמה נפתחת.
3.  המשתמש מצלם -> חזרה לאפליקציה, התמונה מוצגת.
4.  המשתמש מקליד הערה ולוחץ "שמור".
5.  האפליקציה הופכת את התמונה למערך בתים ושומרת ב-SQLite.

## 8. הסבר מפושט
**הסבר במילים פשוטות:**
תארו לעצמכם שאתם **עיתונאים**. `AddToDB` הוא **המחברת והמצלמה** שלכם.
1.  אתם מצלמים תמונה (מצלמה).
2.  אתם כותבים כותרת (EditText).
3.  אתם מדביקים את התמונה במחברת וכותבים את הטקסט לידה (בסיס נתונים SQLite).
ה-`FileProvider` הוא כמו **שומר ראש** שמעביר בבטחה את התמונה מהמצלמה אל המחברת שלכם.
