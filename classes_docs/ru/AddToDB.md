# AddToDB - Описание

## 1. Общая информация
*   **Имя класса:** `AddToDB`
*   **Тип:** Activity (Экран)
*   **Назначение:** Этот экран позволяет пользователю сделать фотографию с помощью камеры устройства, добавить текстовый комментарий и сохранить и то, и другое в локальную базу данных.
*   **Взаимодействие:**
    *   Запрашивает разрешения у системы Android.
    *   Запускает системное приложение "Камера".
    *   Сохраняет данные в `HelperDB` (SQLite).

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `ivImage` | `ImageView` | Для отображения фото, сделанного камерой. | `onCreate()`, `onActivityResult()` |
| `imageFilename` | `String` | Временное имя для файла изображения. | Клик по `ivImage`, `createImageFile()` |
| `imageUri` | `Uri` | "Адрес" файла изображения в системе. | `callCameraApp()`, `onActivityResult()` |
| `helperDB` | `HelperDB` | Соединение с базой данных SQLite. | Клик по `bSave` |
| `db` | `SQLiteDatabase` | Объект для выполнения операций SQL (Вставка). | Клик по `bSave` |
| `etComment` | `EditText` | Текстовое поле для ввода описания пользователем. | `onCreate()`, клик по `bSave` |
| `bSave` | `Button` | Кнопка для сохранения данных в базу. | `onCreate()` |

## 3. Методы класса
### Имя метода: `callCameraApp`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика:**
    1.  Создает Intent для `ACTION_IMAGE_CAPTURE` (открытие камеры).
    2.  Создает временный файл для фото с помощью `createImageFile()`.
    3.  Использует `FileProvider` для получения безопасного URI (адреса) файла.
    4.  Передает этот URI приложению камеры, чтобы оно знало, куда сохранить фото.
    5.  Запускает камеру с помощью `startActivityForResult`.
*   **Когда вызывается:** При нажатии пользователем на `ImageView`.

### Имя метода: `onActivityResult`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `requestCode` | `int` | ID запроса (3 для камеры). |
    | `resultCode` | `int` | Результат: сделал ли пользователь фото или отменил. |
    | `data` | `Intent` | Дополнительные данные (не используются, так как мы используем `imageUri`). |
*   **Логика:** Если пользователь успешно сделал фото, метод устанавливает его в `ivImage` для отображения.

### Имя метода: `bSave.onClick` (внутри `onCreate`)
*   **Логика:**
    1.  Получает текст из `etComment`.
    2.  Получает изображение (bitmap) из `ivImage`.
    3.  Преобразует изображение в массив байтов (`byte[]`) для хранения в БД.
    4.  Открывает базу данных для записи.
    5.  Помещает комментарий и байты изображения в объект `ContentValues`.
    6.  Вставляет строку в таблицу базы данных.
    7.  Закрывает базу данных и завершает работу экрана.

## 4. Жизненный цикл (Activity)
*   **`onCreate()`:** Запрашивает разрешения (Камера, Память) и настраивает слушателей интерфейса.
*   **`onActivityResult()`:** Обрабатывает возврат из приложения камеры для показа снимка.

## 5. Взаимодействие с интерфейсом (UI)
*   **`ImageView`:** Служит кнопкой для открытия камеры и окном предварительного просмотра фото.
*   **`EditText`:** Принимает текст от пользователя.
*   **`Button`:** Запускает процесс сохранения.

## 6. Взаимодействие с другими компонентами
*   **Приложение "Камера":** Внешнее системное приложение.
*   **FileProvider:** Используется для безопасной передачи файлов между приложениями.
*   **HelperDB:** Класс для управления базой данных.

## 7. Общая логика класса
1.  Пользователь дает разрешения.
2.  Пользователь нажимает на иконку изображения -> открывается Камера.
3.  Пользователь делает фото -> возврат в приложение, фото отображается.
4.  Пользователь вводит комментарий и жмет "Сохранить".
5.  Приложение превращает картинку в набор байтов и сохраняет в SQLite.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте, что вы **журналист**. `AddToDB` — это ваш **блокнот и фотоаппарат**.
1.  Вы делаете снимок (Камера).
2.  Вы пишете заголовок (EditText).
3.  Вы вклеиваете фото в блокнот и записываете текст рядом (База данных SQLite).
`FileProvider` — это как **курьер**, который безопасно приносит фотографию от фотоаппарата к вашему блокноту.
