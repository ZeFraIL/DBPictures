# HelperDB - Описание

## 1. Общая информация
*   **Имя класса:** `HelperDB`
*   **Тип:** Обычный класс (наследник `SQLiteOpenHelper`)
*   **Назначение:** Этот класс отвечает за создание и управление локальной базой данных SQLite. Он определяет структуру таблиц, колонок и следит за версией базы данных.
*   **Взаимодействие:** Используется классами `AddToDB` (для сохранения данных) и `ViewFromDB` (для чтения данных).

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `DBNAME` | `String` | Реальное имя файла базы данных на телефоне (`places.db`). | Конструктор `HelperDB` |
| `PLACE_TABLE` | `String` | Имя таблицы (`Places`). | `onCreate`, `AddToDB`, `ViewFromDB` |
| `PLACE_COMMENT`| `String` | Имя текстовой колонки (`About_place`). | `onCreate`, `AddToDB`, `ViewFromDB` |
| `PLACE_PICTURE`| `String` | Имя колонки для изображений (`Place_picture`). | `onCreate`, `AddToDB`, `ViewFromDB` |

*   **Константы:** Эти переменные помечены как `static final`, что означает, что они общие для всего приложения и их нельзя изменить. Это помогает избежать опечаток в SQL-запросах.

## 3. Методы класса
### Имя метода: `HelperDB` (Конструктор)
*   **Тип:** `public`
*   **Логика:** Вызывает родительский конструктор, передавая имя базы данных и версию (1).
*   **Когда вызывается:** При создании нового экземпляра через `new HelperDB(context)`.

### Имя метода: `onCreate`
*   **Тип:** `public`
*   **Возвращаемое значение:** `void` (ничего не возвращает)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `db` | `SQLiteDatabase` | Объект базы данных, в котором будет создана таблица. |
*   **Логика:**
    1.  Собирает строку SQL-команды `CREATE TABLE`.
    2.  Указывает, что колонка `PLACE_COMMENT` имеет тип `TEXT`, а `PLACE_PICTURE` — тип `BLOB`.
    3.  Выполняет команду через `db.execSQL`.
*   **Когда вызывается:** Автоматически системой Android только один раз — когда приложение впервые обращается к базе данных.
*   **Важно знать:** `BLOB` (Binary Large Object) — это специальный тип данных для хранения "сырых" байтов, таких как картинки, видео или музыка.

### Имя метода: `onUpgrade`
*   **Тип:** `public`
*   **Логика:** В данный момент метод пуст. Он предназначен для обновления структуры базы данных (например, добавления новых колонок) в будущих версиях приложения.

## 4. Жизненный цикл
*   **Не является Activity:** Этот класс живет по логике обращения к данным. Он создается, когда нужно что-то записать или прочитать.

## 5. Взаимодействие с интерфейсом (UI)
*   **Отсутствует:** Этот класс работает исключительно "под капотом" (в фоновом режиме) и не имеет графического интерфейса.

## 6. Взаимодействие с другими компонентами
*   **SQLite:** Прямое взаимодействие с движком базы данных Android.

## 7. Общая логика класса
1.  Предоставляет имена таблиц и колонок другим классам.
2.  Гарантирует, что таблица существует в памяти телефона, когда приложению нужно сохранить или прочитать данные.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте `HelperDB` как **архитектора и строителя** склада.
1.  Архитектор придумывает название здания (`DBNAME`).
2.  Рисует план комнаты под названием "Места" (`PLACE_TABLE`).
3.  В комнате делает две полки: одну для блокнотов (`TEXT` — комментарии) и одну для фотоальбомов (`BLOB` — фотографии).
Когда вы впервые запускаете приложение, `HelperDB` строит этот склад, чтобы другие части программы могли класть туда вещи и находить их.
