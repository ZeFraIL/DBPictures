# Info - Описание

## 1. Общая информация
*   **Имя класса:** `Info`
*   **Тип:** Обычный класс (Модель / POJO)
*   **Назначение:** Этот класс является "контейнером" для данных. Он хранит в себе одну запись, состоящую из комментария и изображения (в виде байтов).
*   **Взаимодействие:** Класс `ViewFromDB` создает список объектов `Info` для управления данными, прочитанными из базы данных.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `infoComment` | `String` | Хранит текстовое описание места. | Конструктор, Геттеры/Сеттеры |
| `infoImage` | `byte[]` | Хранит само изображение в виде последовательности чисел (байтов). | Конструктор, Геттеры/Сеттеры |

*   **byte[] (Массив байтов):** Набор маленьких чисел. Так как цифровое изображение — это просто коллекция чисел, представляющих цвета, мы храним его в памяти именно так.

## 3. Методы класса
### Имя метода: `Info` (Конструктор)
*   **Тип:** `public`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `infoComment` | `String` | Текст для сохранения. |
    | `infoImage` | `byte[]` | Байты изображения для сохранения. |
*   **Логика:** Присваивает входящие значения полям класса.
*   **Когда вызывается:** Когда нужно создать в памяти объект для одной записи из базы.

### Имена методов: `getInfoComment` / `setInfoComment`
*   **Назначение:** Стандартные "Геттеры" и "Сеттеры". Они позволяют другим классам безопасно читать или изменять комментарий.

### Имена методов: `getInfoImage` / `setInfoImage`
*   **Назначение:** Стандартные "Геттеры" и "Сеттеры" для данных изображения.

## 4. Жизненный цикл
*   **Обычный объект:** Живет в памяти до тех пор, пока он нужен приложению (обычно пока открыт экран `ViewFromDB`).

## 5. Взаимодействие с интерфейсом (UI)
*   **Напрямую отсутствует:** Он лишь предоставляет данные для элементов интерфейса (например, для всплывающего окна `AlertDialog`).

## 6. Взаимодействие с другими компонентами
*   **ViewFromDB:** Используется внутри списка `ArrayList<Info>`.

## 7. Общая логика класса
1.  Работает как "посылка". Вместо того чтобы передавать строку и массив байтов по отдельности, мы упаковываем их в один объект `Info`.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте класс `Info` как **конверт**.
Внутри конверта лежит **письмо** (текстовый комментарий) и **фотография** (байты изображения).
Когда библиотекарь (`ViewFromDB`) достает данные из хранилища (`HelperDB`), он кладет письмо и фото в этот конверт, чтобы их было удобно переносить внутри приложения.
