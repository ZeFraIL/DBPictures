# MainActivity - Описание

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Экран)
*   **Назначение:** Это "Главное меню" приложения. Оно предоставляет пользователю две основные возможности: добавить новую запись с фотографией или просмотреть существующие записи.
*   **Взаимодействие:** Выступает в роли пульта управления. Запускает `AddToDB` (Добавление), когда пользователь хочет сделать фото, и `ViewFromDB` (Просмотр), когда пользователь хочет увидеть сохраненные данные.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `bAdd` | `Button` | Кнопка для перехода на экран добавления. | `onCreate()` |
| `bView` | `Button` | Кнопка для перехода на экран просмотра. | `onCreate()` |
| `context` | `Context` | Хранит информацию о среде приложения. | `onCreate()`, используется для запуска интентов. |

*   **Context (Контекст):** Представьте это как "паспорт" приложения. Android должен знать, какая часть приложения запрашивает действие.

## 3. Методы класса
### Имя метода: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего не возвращает)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Хранит состояние экрана, если он был закрыт и перезапущен. |
*   **Логика:**
    1.  Вызывает родительский метод `super.onCreate` для стандартной инициализации.
    2.  Устанавливает внешний вид из файла `activity_main.xml`.
    3.  Инициализирует переменную `context`.
    4.  Находит кнопки в макете с помощью `findViewById`.
    5.  Устанавливает "Слушатели" (код, который ждет нажатия) на кнопки.
*   **Когда вызывается:** Автоматически системой Android при запуске приложения.
*   **Важно:** Это входная точка. Если не вызвать `setContentView`, экран будет пустым.

## 4. Жизненный цикл (Activity)
*   **`onCreate()`:** Вызывается при первом создании экрана. Здесь интерфейс связывается с кодом и инициализируется основная логика.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `Button` (bAdd, bView).
*   **Связь:** Через `findViewById(R.id...)`.
*   **События:** `setOnClickListener` обрабатывает нажатия пользователя.

## 6. Взаимодействие с другими компонентами
*   **`Intent` (Интент):** Используется для перехода от `MainActivity` к `AddToDB` или `ViewFromDB`. Интент — это, по сути, "сообщение" системе Android: "Я хочу открыть этот экран".

## 7. Общая логика класса
1.  Приложение запускается.
2.  `MainActivity` загружает две кнопки.
3.  Пользователь нажимает на кнопку.
4.  Приложение отправляет `Intent` для открытия нужного экрана.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте `MainActivity` как **стойку регистрации** в отеле. Когда вы заходите, администратор не дает вам сразу номер, а спрашивает, что вы хотите сделать. У вас есть две кнопки (варианта): "Пройти в фотостудию" (Добавить) или "Пойти в галерею" (Просмотр). `MainActivity` просто указывает вам правильное направление.
