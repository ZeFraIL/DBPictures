# ViewFromDB - Описание

## 1. Общая информация
*   **Имя класса:** `ViewFromDB`
*   **Тип:** Activity (Экран)
*   **Назначение:** Этот экран отображает список всех сохраненных мест. Когда пользователь нажимает на элемент списка, приложение показывает сохраненное фото и комментарий во всплывающем окне.
*   **Взаимодействие:**
    *   Читает данные из `HelperDB`.
    *   Использует класс `Info` для временного хранения данных в памяти.
    *   Заполняет `ListView` заголовками (комментариями).

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `helperDB` | `HelperDB` | Для доступа к базе данных. | `buildList()` |
| `db` | `SQLiteDatabase` | Объект для чтения данных из таблиц. | `buildList()` |
| `lv` | `ListView` | Элемент интерфейса для отображения списка. | `onCreate()`, `buildList()` |
| `all` | `ArrayList<String>` | Список строк-комментариев для показа в ListView. | `buildList()` |
| `allinfo` | `ArrayList<Info>` | Список полных объектов (текст + байты изображения). | `buildList()`, `onItemClick` |
| `adapter` | `ArrayAdapter` | "Мост" между данными и списком ListView. | `buildList()` |
| `context` | `Context` | Среда приложения. | `onCreate()`, `onItemClick` |

## 3. Методы класса
### Имя метода: `buildList`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Логика:**
    1.  Инициализирует списки `all` и `allinfo`.
    2.  Открывает базу данных для чтения.
    3.  Запрашивает таблицу `Places`, чтобы получить все строки.
    4.  Использует `Cursor` (курсор) — это указатель, который переходит от строки к строке.
    5.  Для каждой строки:
        *   Получает комментарий (колонка 0).
        *   Получает байты изображения (колонка 1).
        *   Создает объект `Info` и добавляет его в `allinfo`.
        *   Добавляет текст комментария в список `all`.
    6.  Закрывает базу данных.
    7.  Создает `ArrayAdapter` и подключает его к `ListView`.
*   **Когда вызывается:** Автоматически в `onCreate()`, чтобы сразу показать данные.

### Имя метода: `onItemClick` (внутри `onCreate`)
*   **Логика:**
    1.  Определяет, на какой элемент нажал пользователь.
    2.  Получает соответствующий объект `Info` из списка `allinfo`.
    3.  Преобразует массив байтов из базы данных обратно в картинку (`Bitmap`) с помощью `BitmapFactory`.
    4.  Создает `AlertDialog` (всплывающее окно).
    5.  Устанавливает заголовок окна равным тексту комментария.
    6.  Создает `ImageView` и помещает в него картинку.
    7.  Показывает окно пользователю.
*   **Когда вызывается:** Когда пользователь нажимает на строку в списке.

## 4. Жизненный цикл (Activity)
*   **`onCreate()`:** Инициализирует интерфейс, настраивает слушатель нажатий на список и вызывает `buildList()`.

## 5. Взаимодействие с интерфейсом (UI)
*   **`ListView`:** Отображает строки данных.
*   **`AlertDialog`:** Позволяет просматривать детали (фото) без перехода на другой экран.

## 6. Взаимодействие с другими компонентами
*   **HelperDB:** Для чтения данных из SQLite.
*   **Класс Info:** Служит контейнером для каждой записи.

## 7. Общая логика класса
1.  Экран открывается.
2.  Приложение читает всё из SQLite.
3.  Приложение показывает только комментарии в виде списка.
4.  Пользователь нажимает на комментарий.
5.  Приложение находит "спрятанную" за этим комментарием картинку и показывает её в окне.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте **картотеку в библиотеке**.
1.  `ViewFromDB` — это ящик с карточками.
2.  На каждой карточке виден только заголовок (комментарий).
3.  Когда вы вытаскиваете карточку (нажимаете на пункт списка), библиотекарь приносит вам **саму фотографию** из хранилища (базы данных) и показывает её вам.
`Cursor` (Курсор) — это как ваш **палец**, который ведет по строчкам в книге, пока вы их читаете одну за другой.
