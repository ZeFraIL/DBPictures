## 🎯 מטרת הפרויקט
• **איזו משימה האפליקציה פותרת?**
מאפשרת למשתמשים לצלם תמונות של מקומות ולשמור אותן יחד עם הערות טקסט ישירות לבסיס נתונים מקומי (SQLite) לאחסון וצפייה במצב לא מקוון.
• **מדוע משימה זו חשובה?**
מספקת דרך פשוטה ליצור יומן חזותי אישי או קטלוג של מיקומים ללא צורך באחסון בענן או בחיבור לאינטרנט.
• **קהל יעד:**
מטיילים, חוקרים, או כל מי שרוצה לתעד במהירות מיקומים עם תמונות והערות.

## 🧠 ארכיטקטורה כללית
• **הגישה שנבחרה:**
MVC (Model-View-Controller) - הגישה הקלאסית לפרויקטי Java פשוטים באנדרואיד.
• **מדוע היא נבחרה?**
בהתחשב בפשטות הפרויקט, MVC מספק מבנה ברור שבו ה-Activities מנהלות את לוגיקת הממשק והבקרה, בעוד המחלקות `HelperDB` ו-`Info` מייצגות את שכבת הנתונים.
• **מרכיבי המערכת העיקריים:**
- `MainActivity`: נקודת כניסה וניווט.
- `AddToDB`: בקר לצילום תמונות ושמירת נתונים.
- `ViewFromDB`: בקר לשליפה והצגת נתונים.
- `HelperDB`: ניהול בסיס נתונים SQLite.
- `Info`: מחלקת מודל נתונים.

## 🧩 תיאור מחלקות מפורט
### 📌 מחלקה: MainActivity
**תפקיד:** מסך ראשי.
**אחריות:** מספקת ניווט למודולים של "הוספה" ו"צפייה".
**שיטות עיקריות:**
- `onCreate()`: אתחול כפתורים והגדרת מאזיני לחיצה לניווט.
**אינטראקציה עם מחלקות אחרות:** מפעילה את `AddToDB` ו-`ViewFromDB` באמצעות Intents.

### 📌 מחלקה: AddToDB
**תפקיד:** מסך הזנת נתונים.
**אחריות:** מנהלת אינטראקציה עם המצלמה, קלט משתמש (EditText) ושמירה לבסיס הנתונים.
**שיטות עיקריות:**
- `callCameraApp()`: מכינה את הקובץ ומפעילה את המצלמה.
- `onActivityResult()`: מקבלת את התמונה ומציגה אותה ב-ImageView.
- `bSave.onClick()`: דוחסת את התמונה למערך בתים ושומרת ל-SQLite.

### 📌 מחלקה: ViewFromDB
**תפקיד:** מסך צפייה בנתונים.
**אחריות:** שולפת רשומות מ-SQLite ומציגה אותן ב-ListView.
**שיטות עיקריות:**
- `buildList()`: שואלת את בסיס הנתונים וממלאת את מתאם הרשימה (Adapter).
- `onItemClick()`: מציגה תמונה בגודל מלא והערה ב-AlertDialog.

### 📌 מחלקה: Repository / Data Layer (HelperDB)
**תפקיד:** שכבת נתונים.
**אחריות:** יצירה וניהול של סכימת בסיס הנתונים SQLite (טבלת `Places`).

## 🔄 דיאגרמת פעולה של האפליקציה
1. המשתמש פותח את האפליקציה (`MainActivity`).
2. המשתמש לוחץ על "Add" -> `AddToDB` נפתח.
3. המשתמש לוחץ על `ImageView` -> המצלמה נפתחת.
4. המשתמש מצלם תמונה -> התמונה מוצגת ב-`ImageView`.
5. המשתמש מזין הערה ולוחץ על "Save" -> הנתונים נשמרים ב-SQLite.
6. המשתמש חוזר ולוחץ על "View" -> `ViewFromDB` מציג את הרשימה.
7. המשתמש לוחץ על פריט ברשימה -> `AlertDialog` מציג את התמונה והטקסט.
