## 🎯 Цель проекта
•	**Какую задачу решает приложение:**
Позволяет пользователю делать снимки мест и сохранять их вместе с текстовыми комментариями в локальную базу данных SQLite для автономного хранения и просмотра.
•	**Почему эта задача важна:**
Обеспечивает простой способ создания личного визуального дневника или каталога мест без необходимости использования облачных хранилищ или интернет-соединения.
•	**Целевая аудитория:**
Путешественники, исследователи или любые пользователи, которым нужно быстро зафиксировать местоположение с фото и заметкой.

## 🧠 Общая архитектура
•	**Выбранный подход:**
MVC (Model-View-Controller) — классический подход для простых Java-проектов под Android.
•	**Почему выбран именно он:**
Для текущего масштаба проекта MVC обеспечивает понятную структуру, где Activity управляют логикой UI, а классы `HelperDB` и `Info` отвечают за уровень данных.
•	**Основные компоненты системы:**
–	`MainActivity`: Точка входа и навигация.
–	`AddToDB`: Контроллер для захвата фото и сохранения данных.
–	`ViewFromDB`: Контроллер для извлечения и отображения данных.
–	`HelperDB`: Управление базой данных SQLite.
–	`Info`: Модель данных.

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
**Роль:** Главный экран.
**Ответственность:** Навигация между модулями добавления и просмотра.
**Основные методы:**
– `onCreate()` — инициализация кнопок и установка слушателей кликов.
**Взаимодействие с другими классами:** Запускает `AddToDB` и `ViewFromDB` через Intent.

### 📌 Класс: AddToDB
**Роль:** Экран добавления данных.
**Ответственность:** Взаимодействие с камерой, ввод текста и сохранение в БД.
**Основные методы:**
– `callCameraApp()` — подготовка файла и вызов камеры.
– `onActivityResult()` — получение фото и отображение в ImageView.
– `bSave.onClick()` — сжатие фото в массив байтов и сохранение в SQLite.

### 📌 Класс: ViewFromDB
**Роль:** Экран просмотра.
**Ответственность:** Загрузка записей из БД и их отображение в списке.
**Основные методы:**
– `buildList()` — запрос к БД и заполнение адаптера списка.
– `onItemClick()` — открытие диалога с полноразмерным фото и текстом.

### 📌 Класс: Repository / Data Layer (HelperDB)
**Роль:** Слой данных.
**Ответственность:** Создание и управление схемой базы данных SQLite (таблица `Places`).

## 🔄 Диаграмма работы приложения
1. Пользователь открывает приложение (`MainActivity`).
2. Нажимает "Add" -> открывается `AddToDB`.
3. Нажимает на `ImageView` -> запускается камера.
4. Делает фото -> оно отображается на экране.
5. Вводит комментарий и жмет "Save" -> данные уходят в SQLite.
6. Возвращается и жмет "View" -> `ViewFromDB` показывает список.
7. Нажимает на элемент списка -> открывается `AlertDialog` с фото и текстом.

## ⚙️ Работа с потоками
•	**Используются ли:**
– Основной UI-поток для всех операций.
•	**Почему выбран этот способ:** Для небольших локальных БД в учебных или простых проектах это допустимо, хотя для больших данных стоит использовать `Executors` или `Coroutines`.
•	**Как предотвращаются:**
– Зависания (ANR): Операции с БД выполняются быстро из-за малого объема.
– Утечки памяти: Использование стандартных жизненных циклов Activity.

## 💾 Работа с данными
•	**Где хранятся данные:** Локальная база данных SQLite (`places.db`).
•	**Почему выбран этот способ:** Надежность, структурированность и полная автономность.
•	**Как обеспечивается:**
– Сохранность: SQL-транзакции и корректное закрытие БД.
– Корректность: Использование BLOB для хранения изображений в связке с текстом.
